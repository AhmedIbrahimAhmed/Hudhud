# سكربت العرض التقديمي - منصة هدهد
## منصة الصحافة الفلسطينية الذكية

---

## الجزء الأول: للصحفيين (غير التقنيين)

### المقدمة - الفكرة الأساسية

**المشكلة التي نحلها:**
الصحفيون الفلسطينيون يواجهون تحديات يومية في التحقق من المعلومات، كتابة المقالات بدقة، والتعاون مع الفرق بشكل آمن. المنصة توفر لهم أدوات متكاملة في مكان واحد.

**المنصة هي:**
- مساعد ذكي للكتابة والتحرير
- أداة للتحقق من الصور والفيديوهات
- نظام لإدارة المهام والمشاريع الجماعية
- منصة آمنة للتعاون بين الفرق

---

### شرح الأقسام الرئيسية

#### 1. صفحة الفريق (Team)
**ماذا تفعل؟**
- إنشاء فريق صحفي أو الانضمام لفريق موجود
- دعوة أعضاء جدد للفريق
- إدارة صلاحيات الأعضاء (قائد/عضو)

**كيف تساعد الصحفي؟**
- تنظيم العمل الجماعي في مكان واحد
- تتبع مهام كل عضو
- التواصل المباشر داخل الفريق

---

#### 2. المهام والتقويم (Tasks/Calendar)
**ماذا تفعل؟**
- إنشاء مهام شخصية أو مهام للفريق
- تحديد مواعيد استحقاق للمهام
- رفع ملفات للمهام
- عرض المهام في تقويم شهري

**كيف تساعد الصحفي؟**
- تنظيم المواعيد والمواعيد النهائية
- تتبع تقدم العمل
- تنبيهات للمهام القادمة
- سهولة مشاركة الملفات مع الفريق

---

#### 3. كتابة المقالات (Article Writing)
**ماذا تفعل؟**
- محرر نصوص ذكي للكتابة
- تصحيح أخطاء اللغة العربية تلقائياً
- اقتراحات لتحسين الأسلوب
- مساعد ذكي للإجابة على الأسئلة أثناء الكتابة
- حفظ المسودات تلقائياً

**كيف تساعد الصحفي؟**
- توفير الوقت في التدقيق اللغوي
- تحسين جودة المقالات
- الحصول على مساعدة فورية أثناء الكتابة
- عدم فقدان العمل (حفظ تلقائي)

---

#### 4. أدوات الصوت (Voice Tools)
**ماذا تفعل؟**
- تحويل الصوت إلى نص (تسجيل المقابلات)
- تحويل النص إلى صوت (قراءة المقالات)
- دعم اللغة العربية

**كيف تساعد الصحفي؟**
- تسجيل المقابلات وتحويلها لنص فوراً
- قراءة المقالات صوتياً
- توفير وقت الكتابة اليدوية

---

#### 5. فاحص الروابط (Link Checker)
**ماذا تفعل؟**
- التحقق من سلامة الروابط
- كشف الروابط الاحتيالية
- تحليل مصداقية المواقع

**كيف تساعد الصحفي؟**
- حماية من الروابط الضارة
- التحقق من مصداقية المصادر
- تجنب نشر معلومات من مواقع مشبوهة

---

#### 6. التحقق من الصور (Image Verification)
**ماذا تفعل؟**
- البحث العكسي عن الصور (أين ظهرت على الإنترنت)
- كشف الصور المزيفة بالذكاء الاصطناعي
- تحديد ما إذا كانت الصورة حقيقية أو مولدة بالذكاء الاصطناعي

**كيف تساعد الصحفي؟**
- التحقق من مصداقية الصور المنشورة
- كشف التزييف والتعديل على الصور
- التأكد من أن الصورة حقيقية وليست مولدة

---

#### 7. محرر الصور (Image Editor)
**ماذا تفعل؟**
- أدوات أساسية لتعديل الصور
- قص وتغيير الحجم
- تحسين الإضاءة والألوان

**كيف تساعد الصحفي؟**
- تحسين جودة الصور قبل النشر
- إضافة علامات مائية (إذا لزم)
- تجهيز الصور للنشر

---

#### 8. فحص الفيديو (Video Check)
**ماذا تفعل؟**
- كشف التزييف في الفيديوهات (Deepfake)
- تحليل محتوى الفيديو

**كيف تساعد الصحفي؟**
- التحقق من مصداقية الفيديوهات
- كشف التعديلات الاحتيالية
- حماية من نشر محتوى مزيف

---

#### 9. المسودات (Drafts)
**ماذا تفعل؟**
- عرض جميع المقالات المحفوظة
- فتح مسودة للتعديل
- حذف المسودات غير المرغوبة

**كيف تساعد الصحفي؟**
- الوصول السريع لجميع الأعمال
- إدارة المشاريع المتعددة
- عدم فقدان أي عمل

---

## الجزء الثاني: للمطورين (تقني)

### نظرة عامة على البنية المعمارية

**البنية:**
- **Frontend:** React + Vite
- **Backend:** Node.js + Express
- **Database:** SQLite (better-sqlite3)
- **Authentication:** JWT (JSON Web Tokens)
- **File Storage:** Cloudinary (للصور والملفات)
- **Real-time:** WebSocket (للنسخ الصوتي المباشر)

---

### شرح الميزات التقنية

#### 1. نظام المصادقة (Authentication)
**كيف يعمل؟**
- التسجيل والدخول عبر JWT
- التخزين الآمن لكلمات المرور (bcrypt)
- حماية جميع المسارات بـ middleware
- انتهاء صلاحية الجلسة تلقائياً

**التدفق:**
1. المستخدم يسجل الدخول → يُرسل البريد وكلمة المرور
2. الخادم يتحقق من البيانات → يُنشئ JWT token
3. العميل يحفظ الـ token في localStorage
4. كل طلب API يحمل الـ token في header
5. الخادم يتحقق من الـ token قبل معالجة الطلب

---

#### 2. معالجة المقالات بالذكاء الاصطناعي
**كيف يعمل؟**
- استخدام Google Gemini API للتحليل اللغوي
- كشف المصطلحات الخاصة
- تصحيح الأخطاء الإملائية والنحوية
- تحليل SEO
- الاعتدال (Moderation) للمحتوى

**التدفق:**
1. المستخدم يكتب المقال → يضغط "معالجة"
2. الخادم يرسل النص لـ Gemini API
3. Gemini يُرجع التصحيحات والتحليلات
4. الخادم يُنظم النتائج ويعرضها للمستخدم
5. المستخدم يقبل/يرفض كل تصحيح

---

#### 3. المساعد الذكي للدردشة (Chat Assistant)
**كيف يعمل؟**
- دعم نماذج متعددة (Gemini, OpenRouter)
- حفظ سجل المحادثة مع كل مقال
- إرسال سجل المحادثة للسياق

**التدفق:**
1. المستخدم يسأل سؤالاً
2. الخادم يرسل السؤال + سجل المحادثة للـ AI
3. AI يُرجع الإجابة
4. الإجابة تُعرض وتُحفظ في سجل المقال

---

#### 4. التحقق من الصور
**كيف يعمل؟**
- **البحث العكسي:** محرك بحث صور خاص (مملوك)
- **كشف الذكاء الاصطناعي:** Sightengine API
- **كشف Deepfake:** ScamAI API

**التدفق:**
1. المستخدم يرفع صورة أو يُدخل رابط
2. إذا كان رابط → الخادم يُحمّل الصورة
3. الصورة تُرفع لمضيف مؤقت للحصول على رابط عام
4. البحث العكسي: يُرسل الرابط للمحرك → يُرجع النتائج
5. كشف AI: الصورة تُرفع مباشرة لـ Sightengine/ScamAI
6. النتائج تُعرض للمستخدم

---

#### 5. النسخ الصوتي (Transcription)
**كيف يعمل؟**
- استخدام Deepgram API
- WebSocket للنسخ المباشر
- دعم اللغة العربية

**التدفق:**
1. المستخدم يبدأ التسجيل
2. الصوت يُرسل عبر WebSocket
3. Deepgram يُحوّل الصوت لنص مباشرة
4. النص يُعرض للمستخدم أثناء التسجيل

---

#### 6. تحويل النص إلى صوت (Text-to-Speech)
**كيف يعمل؟**
- استخدام Microsoft Edge TTS API (مجاني)
- دعم أصوات عربية متعددة

**التدفق:**
1. المستخدم يختار النص والصوت
2. الخادم يرسل الطلب لـ Edge TTS
3. Edge يُرجع ملف صوتي (MP3)
4. الملف يُشغّل للمستخدم

---

#### 7. إدارة الفرق والمهام
**كيف يعمل؟**
- نظام صلاحيات (قائد/عضو)
- قاعدة بيانات علائقية (SQLite)
- Foreign keys للحماية

**التدفق:**
1. إنشاء فريق → يُنشأ في قاعدة البيانات
2. دعوة عضو → يُرسل إشعار
3. العضو يقبل → يُضاف لـ team_members
4. المهام تُربط بالفريق والعضو
5. عند حذف فريق → تُحذف جميع البيانات المرتبطة (CASCADE)

---

#### 8. نظام الإشعارات
**كيف يعمل؟**
- إشعارات للدعوات والمهام والرسائل
- عداد للإشعارات غير المقروءة
- تحديث تلقائي (polling كل 4 ثواني)

**التدفق:**
1. حدث يحدث (دعوة، مهمة، رسالة)
2. إشعار يُنشأ في قاعدة البيانات
3. العميل يستعلم عن الإشعارات بشكل دوري
4. الإشعارات الجديدة تُعرض للمستخدم

---

### التقنيات المستخدمة

**Frontend:**
- React 18
- Vite (بناء سريع)
- React Router (التوجيه)
- Axios (HTTP client)
- Tailwind CSS (التنسيق)
- IBM Plex Sans Arabic (الخط)

**Backend:**
- Node.js
- Express.js
- better-sqlite3 (قاعدة البيانات)
- bcrypt (تشفير كلمات المرور)
- jsonwebtoken (JWT)
- multer (رفع الملفات)
- ws (WebSocket)
- dotenv (إدارة المتغيرات البيئية)

**External APIs:**
- Google Gemini (معالجة النصوص)
- OpenRouter (نماذج AI بديلة)
- Deepgram (النسخ الصوتي)
- Microsoft Edge TTS (تحويل نص لصوت)
- Sightengine (كشف AI في الصور)
- ScamAI (كشف Deepfake)
- محرك بحث صور خاص (البحث العكسي)
- Cloudinary (تخزين الملفات)

---

### التحديات التقنية والحلول

#### التحدي 1: معالجة الأخطاء في AI
**المشكلة:** AI APIs قد تُرجع JSON غير صالح
**الحل:** معالجة الأخطاء مع إعادة المحاولة، fallback لنماذج بديلة

#### التحدي 2: رفع الملفات الكبيرة
**المشكلة:** حدود حجم الملفات
**الحل:** استخدام Cloudinary للملفات الكبيرة، multer للملفات الصغيرة

#### التحدي 3: الأمان
**المشكلة:** حماية البيانات الحساسة
**الحل:** JWT للتوثيق، bcrypt لكلمات المرور، prepared statements لـ SQL

#### التحدي 4: الأداء
**المشكلة:** بطء استجابة AI APIs
**الحل:** معالجة غير متزامنة، timeouts، رسائل خطأ واضحة

#### التحدي 5: دعم العربية
**المشكلة:** بعض APIs لا تدعم العربية جيداً
**الحل:** استخدام APIs مخصصة للعربية، اختبار شامل

---

### تدفق البيانات بين المكونات

```
المستخدم ←→ Frontend (React)
              ↓
              HTTP/WebSocket
              ↓
         Backend (Express)
              ↓
         SQLite Database
              ↓
    External APIs (Gemini, Deepgram, etc.)
```

**مثال: معالجة مقال**
1. Frontend يرسل النص لـ Backend
2. Backend يحفظ في SQLite (مسودة)
3. Backend يرسل لـ Gemini API
4. Gemini يُرجع التصحيحات
5. Backend يحفظ النتائج في SQLite
6. Backend يُرجع النتائج لـ Frontend
7. Frontend يعرض التصحيحات

## الجزء الثالث: للمقدم (سكربت العرض)

### الافتتاحية

"السلام عليكم ورحمة الله وبركاتهم،

أنا سأقدم لكم اليوم منصة 'هدهد' - منصة ذكية للصحافة الفلسطينية.

هدهد ليست مجرد موقع، بل هي مساعد ذكي متكامل للصحفيين الفلسطينيين، تجمع بين أدوات الكتابة الذكية، والتحقق من المعلومات، والتعاون الآمن بين الفرق."

---

### القسم الأول: المشكلة والحل

"لنبدأ بالمشكلة: الصحفيون الفلسطينيون يواجهون تحديات يومية - كيف نتأكد من صحة الصور؟ كيف نكتب مقالات خالية من الأخطاء؟ كيف نتعاون مع الفرق بأمان؟

هدهد تحل هذه المشكلة بتوفير كل هذه الأدوات في منصة واحدة آمنة وسهلة الاستخدام."

---

### القسم الثاني: جولة في المنصة (للصحفيين)

"دعونا نرى ماذا تقدم هدهد للصحفيين:

**أولاً: الفريق والمهام**
- إنشاء فريق وإدارة أعضائه
- تتبع المهام والمواعيد
- تقويم لتنظيم العمل

**ثانياً: كتابة المقالات**
- محرر ذكي يصحح الأخطاء تلقائياً
- مساعد ذكي يجيب على أسئلتك أثناء الكتابة
- حفظ تلقائي للمسودات

**ثالثاً: أدوات الصوت**
- تحويل المقابلات المسجلة إلى نص فوراً
- قراءة المقالات صوتياً

**رابعاً: التحقق من المعلومات**
- فاحص روابط يكشف الروابط الاحتيالية
- التحقق من الصور - هل هي حقيقية أم مزيفة؟
- فحص الفيديوهات لكشف التزييف

كل هذه الأدوات في مكان واحد، سهلة الاستخدام، ومصممة خصيصاً للصحفيين."

---

### القسم الثالث: الجانب التقني (للمطورين)

"الآن لننتقل للجانب التقني:

**البنية المعمارية:**
- Frontend: React مع Vite للأداء السريع
- Backend: Node.js و Express
- Database: SQLite لسهولة النشر
- Authentication: JWT للأمان

**الميزات التقنية:**
- معالجة المقالات بالذكاء الاصطناعي (Google Gemini)
- البحث العكسي للصور باستخدام محرك بحث خاص
- كشف الصور المزيفة (Sightengine و ScamAI)
- النسخ الصوتي المباشر (Deepgram WebSocket)
- تحويل النص إلى صوت (Microsoft Edge TTS)

**التحديات والحلول:**
- معالجة أخطاء AI APIs مع fallback
- حماية البيانات بـ JWT و bcrypt
- دعم كامل للغة العربية
- أداء محسن مع معالجة غير متزامنة

التدفق: المستخدم ←→ Frontend ←→ Backend ←→ Database ←→ External APIs"

---

### القسم الرابع: الخاتمة

"هدهد تجمع بين البساطة للصحفيين والقوة للمطورين.

للصحفيين: أدوات سهلة تساعد في العمل اليومي.
للمطورين: بنية معمارية نظيفة وقابلة للتوسع.

شكراً لاستماعكم، وأنا جاهز للإجابة على أي أسئلة."

## أسئلة وأجوبة محتملة

### سؤال: كيف تضمن أمان البيانات؟
**جواب:**
- جميع كلمات المرور مُشفّرة بـ bcrypt
- JWT tokens للتوثيق
- Prepared statements لمنع SQL Injection
- CORS مُكوّن بشكل صحيح
- مسارات API محمية بـ middleware

---

### سؤال: ماذا يحدث إذا فشل AI API؟
**جواب:**
- معالجة أخطاء شاملة
- Fallback لنماذج AI بديلة (OpenRouter)
- رسائل خطأ واضحة للمستخدم
- إعادة المحاولة التلقائية (retry)

---

### سؤال: كيف تدعم المنصة اللغة العربية؟
**جواب:**
- خط IBM Plex Sans Arabic
- APIs مختارة تدعم العربية (Gemini, Deepgram, Edge TTS)
- واجهة مستخدم بالكامل بالعربية
- اختبار شامل للعربية

---

### سؤال: كيف يعمل البحث العكسي للصور؟
**جواب:**
- نستخدم محرك بحث صور خاص (مملوك)
- الصورة تُرفع لمضيف مؤقت للحصول على رابط عام
- الرابط يُرسل للمحرك
- المحرك يُرجع المواقع التي ظهرت فيها الصورة
- النتائج تُعرض للمستخدم

---

### سؤال: كيف تُحفظ المسودات؟
**جواب:**
- حفظ تلقائي كل 1.2 ثانية
- حفظ في قاعدة البيانات (SQLite)
- ربط المسودة بالمستخدم (user_id)
- استعادة المسودة عند إعادة التحميل

---

### سؤال: كيف يعمل نظام الفرق؟
**جواب:**
- قاعدة بيانات علائقية (SQLite)
- Foreign keys للحماية
- CASCADE delete عند حذف فريق
- صلاحيات: قائد (يمكن حذف الفريق) وعضو (لا يمكن)
- إشعارات للدعوات

---

### سؤال: ما هي خطط التطوير المستقبلية؟
**جواب:**
- إضافة المزيد من نماذج AI
- تحسين دعم اللغات الأخرى
- تطبيق موبايل
- ميزات تعاونية متقدمة
- تحليلات وإحصائيات

---

### سؤال: كيف يمكن نشر المنصة؟
**جواب:**
- Backend: يمكن نشرها على أي Node.js host (Vercel, Railway, Heroku)
- Database: SQLite ملف واحد، سهل النسخ الاحتياطي
- Frontend: Vite يُنتج static files، يمكن نشرها على أي CDN
- Environment variables: كل المفاتيح في .env

---

### سؤال: كيف تُعالج الملفات الكبيرة؟
**جواب:**
- الملفات الكبيرة تُرفع لـ Cloudinary
- الملفات الصغيرة تُحفظ محلياً
- multer للتعامل مع multipart/form-data
- حدود حجم: 4MB للصور الشخصية، أكبر للملفات الأخرى

---

### سؤال: كيف يعمل WebSocket؟
**جواب:**
- للنسخ الصوتي المباشر فقط
- الصوت يُرسل كـ stream
- Deepgram يُحوّل الصوت لنص في الوقت الحقيقي
- النص يُعرض للمستخدم أثناء التسجيل

- ركّز على القيمة المضافة
